Section 1 — Architecture tổng quan

FRONTEND (Next.js)
 ├─ /studio    ← Wizard UI 6-step per video session
 ├─ /sessions  ← Quản lý tất cả sessions (draft/done)
 └─ /schedule  ← Lịch đăng, calendar view, TikTok status

BACKEND (FastAPI — Python)
 ├─ Pipeline Engine  ← orchestrate 6 bước per session
 ├─ Session Manager  ← save/load state từng bước
 ├─ Asset Manager   ← quản lý Milo image library
 ├─ TikTok Client   ← upload + schedule post
 └─ AI Chat Handler  ← relay message tới Gemini

DATA (SQLite)
 ├─ sessions   ← id, title, step, status, created_at
 ├─ scenes    ← session_id, order, script, image_path, video_path, approved
 └─ schedule  ← session_id, post_time, tiktok_id, status

EXTERNAL (free)
 ├─ Gemini 2.0 Flash API  ← script gen + chat (1500 req/ngày free)
 ├─ Edge-TTS             ← voice VI/EN ($0)
 ├─ FFmpeg               ← video assembly local ($0)
 └─ TikTok Content API   ← publish + schedule ($0)

Section 2 — 6-Step Pipeline per Video

BƯỚC 1
📝 Kịch bản
Input: topic/keyword
AI gen: hook + nội dung + CTA affiliate + hashtag
User: edit trực tiếp trong editor
Output: script text + metadata
BƯỚC 2
🎬 Phân cảnh
AI tự split script → scenes (3–8 cảnh)
Mỗi cảnh: text + timestamp + emotion tag
User: reorder, merge, split, edit text
Output: scene list với emotion labels
BƯỚC 3
🖼 Chọn ảnh Milo
AI match emotion tag → ảnh trong library
User: approve / chọn lại từ gallery
Background: auto-gen gradient hoặc chọn từ pack
Output: image_path per scene
BƯỚC 4
🎥 Video từng cảnh
Edge-TTS → audio per scene
FFmpeg: ken-burns effect + caption overlay
User: preview từng clip, redo nếu cần
Output: scene_N.mp4 per scene
BƯỚC 5
✂️ Ghép video
FFmpeg concat tất cả scene clips
+ fade transitions
+ BGM (Pixabay free music)
User: preview full video, chỉnh âm lượng BGM
Output: final_video.mp4
BƯỚC 6
🚀 Xuất & Đăng
AI gen caption VI/EN + hashtag pack
User: set ngày giờ đăng
TikTok API: upload + schedule
Output: session → status "published"

Section 3 — Session State Machine

draft → step_1 → step_2 → step_3 → step_4 → step_5 → step_6 → scheduled → published
                            ↑_______________________________↑
                            có thể back bất kỳ bước nào

State lưu trong SQLite → tắt máy, quay lại tiếp tục đúng bước đang dở.
