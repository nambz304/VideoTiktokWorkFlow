Giao diện kiểm soát pipeline — chọn hướng nào?

Yêu cầu: thấy flow đang chạy, review output từng bước, approve/reject trước khi tiếp

A

Streamlit (Python web app)

Step 2/5 — Scene Images ████████░░ 80%
▶ Scene 1: [🖼 ảnh Milo tay vẫy] [✓ OK] [✗ Redo]
▶ Scene 2: [🖼 Milo cầm rau củ] [✓ OK] [✗ Redo]
▶ Scene 3: generating...

Pros

  • Pure Python, dễ integrate với pipeline code
  • Show ảnh/video preview trực tiếp
  • Button approve/reject mỗi scene dễ dàng
  • Progress bar, live log real-time
  • CV AI engineering: rất đẹp
  • Deploy local hoặc cloud miễn phí (Streamlit Cloud)

Cons

  • Cần code UI, không drag-drop

Cost $0 · Skill Python

B

n8n (Visual workflow, no-code)

[Script Gen] →✅ [Scene Split] →✅ [Image Pick] →⏸ WAIT APPROVAL
                                                ↓ user clicks Resume
                                  [Video Gen] → [Merge] → [Post]

Pros

  • Drag-drop node workflow, visual đẹp
  • Built-in wait/approval node
  • Self-hosted free
  • Tích hợp TikTok API sẵn

Cons

  • Preview ảnh/video trong n8n kém
  • Review output từng scene không smooth
  • Ít "code" → CV AI kém ấn tượng hơn

Cost $0 · Skill No-code

C

⭐ Streamlit Dashboard + n8n Orchestrator

n8n: trigger → gọi Python pipeline → pause
Streamlit: hiện output đẹp → user approve
n8n: nhận approve signal → chạy tiếp

Pros

  • n8n lo scheduling + orchestration
  • Streamlit lo review UI (ảnh/video đẹp)
  • Tách biệt rõ: logic vs UI
  • CV cực đẹp: "built multi-agent pipeline với custom UI"

Cons

  • Phức tạp hơn A, cần wire 2 system
  • Setup 3-5 ngày

Cost $0 · Skill Python + n8n

D

Gradio (ML-native UI)

Tab 1: Script review
Tab 2: Scene images → approve
Tab 3: Video preview → publish

Pros

  • Rất quen trong AI/ML community
  • Gallery, video player built-in
  • Deploy lên HuggingFace Spaces free

Cons

  • Stateful pipeline control kém hơn Streamlit
  • UX không mượt bằng

Cost $0 · Skill Python

Tôi khuyến nghị C cho CV, A nếu muốn đơn giản hơn.
